# 급할땐 서비스 기획서

버전: v0.1  
작성일: 2026-05-18  
단계: Phase 1 (MVP)

---

## 1. 서비스 개요

**급할땐**은 외출 중 급한 상황에서 가장 가까운 공중화장실을 빠르게 찾아주는 앱입니다.

- **서비스명**: 급할땐 (Geuphalttaen)
- **플랫폼**: iOS / Android (Expo React Native)
- **핵심 가치**: 로그인 없이 즉시 사용 가능, 공공데이터 기반 신뢰성, 사용자 제보로 데이터 확장

---

## 2. 핵심 타겟

- 외출 중 화장실이 급한 모든 사람 (연령·성별 무관)
- 특히 낯선 동네를 여행·출장 중인 사용자
- 장애인 화장실·가족 화장실이 필요한 사용자

---

## 3. 핵심 가치 및 차별점

| 항목 | 설명 |
|------|------|
| 즉시성 | 앱 진입 즉시 현재 위치 기반 지도 표시 — 로그인 없이 바로 사용 |
| 신뢰성 | 행정안전부 공공데이터(공공데이터포털)를 주기적으로 동기화 |
| 확장성 | 로그인 사용자의 위치 제보로 공식 데이터에 없는 비공용 화장실까지 커버 |
| 상세 정보 | 남/여/장애인/가족 화장실 구분, 개방 여부 확인 가능 |

---

## 4. 사용자 시나리오

### 4.1 비로그인 조회 플로우

```
앱 실행
  → 위치 권한 요청
    → 현재 위치 기반 지도에 근처 화장실 마커 표시
      → 마커 또는 목록 아이템 탭
        → 화장실 상세 정보 확인 (이름, 주소, 남/여/장애인/가족 화장실 여부)
          → 지도 앱 연결로 길 안내
```

### 4.2 로그인 후 제보 플로우

```
Kakao / Apple 소셜 로그인
  → 화면 하단의 "제보하기" 버튼 탭
    → 지도에서 위치 핀 지정 또는 주소 검색
      → 화장실 정보 입력 (이름, 남/여/장애인/가족 여부)
        → 제보 완료 (관리자 검토 후 ACTIVE 전환 시 지도에 노출)
```

## 5. 기능 목록 (Phase 1 MVP)

### 5.1 핵심 기능

| 기능 | 설명 | 인증 |
|------|------|------|
| 근처 화장실 지도 표시 | 현재 위치 반경 N미터 내 화장실 마커 표시 | 불필요 |
| 화장실 목록 조회 | 거리순 정렬, 거리/이름/구분 표시 | 불필요 |
| 화장실 상세 정보 | 이름, 주소, 남/여/장애인/가족 화장실 여부 | 불필요 |
| Kakao 로그인 | Kakao OAuth2 → 서버 JWT 발급 | - |
| Apple 로그인 | Apple Sign In → 서버 JWT 발급 | - |
| 화장실 위치 제보 | 비공용 포함, 관리자 승인 후 노출 | 필요 |
| 공공데이터 동기화 | 행정안전부 API 주기적 수집 (`@Scheduled`) | - |

### 5.2 기술적 요구사항

- 화장실 검색 API: 위도·경도·반경(m)을 파라미터로 받아 PostGIS 없이 Haversine 공식으로 거리 계산
- 제보 상태: PENDING → (관리자 승인) → ACTIVE / REJECTED
- JWT: Access Token 1시간, Refresh Token 14일, Refresh는 Redis에 저장
- 공공데이터 동기화: 주 1회 `@Scheduled`, 중복 체감 최소화 (address + name 기준 upsert)

  - URL: https://www.data.go.kr/data/15012752/openapi.do
  - 인코딩: UTF-8

## 8. API 명세 (주요 엔드포인트)

### 8.1 인증

| Method | Path | 설명 | 요청 |
|--------|------|------|------|
| POST | `/api/v1/auth/login` | OAuth 로그인 (Kakao/Apple) | `{ provider, accessToken }` |
| POST | `/api/v1/auth/refresh` | Access Token 갱신 | `{ refreshToken }` |
| POST | `/api/v1/auth/logout` | 로그아웃 (Refresh 무효화) | Bearer JWT |

### 8.2 화장실

| Method | Path | Auth | 설명 | 주요 파라미터 |
|--------|------|------|------|------|
| GET | `/api/v1/toilets` | 없음 | 근처 화장실 검색 | `lat`, `lng`, `radiusMeters` |
| GET | `/api/v1/toilets/{id}` | 없음 | 화장실 상세 조회 | - |
| POST | `/api/v1/toilets/report` | Bearer JWT | 화장실 위치 제보 | `name`, `address`, `lat`, `lng`, `male`, `female`, `disabled`, `familyRoom` |

### 8.3 공통 응답 포맷

```json
{
  "success": true,
  "data": { ... },
  "error": null
}
```

에러 시:
```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "TOILET_NOT_FOUND",
    "message": "화장실 정보를 찾을 수 없습니다."
  }
}
```

## 9. DB 모델

### 9.1 users 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT (PK) | 자동 증가 |
| provider | VARCHAR(10) | KAKAO / APPLE |
| provider_id | VARCHAR(255) | OAuth 제공자의 사용자 ID |
| nickname | VARCHAR(50) | 닉네임 |
| created_at | DATETIME | 생성일 |
| updated_at | DATETIME | 수정일 |

인덱스: UNIQUE(provider, provider_id)

### 9.2 toilets 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT (PK) | 자동 증가 |
| name | VARCHAR(255) | 화장실 이름 |
| address | VARCHAR(500) | 도로명 주소 |
| lat | DOUBLE | 위도 |
| lng | DOUBLE | 경도 |
| is_public | TINYINT(1) | 공공화장실 여부 |
| male | TINYINT(1) | 남성 화장실 여부 |
| female | TINYINT(1) | 여성 화장실 여부 |
| disabled | TINYINT(1) | 장애인 화장실 여부 |
| family_room | TINYINT(1) | 가족 화장실 여부 |
| reported_by | BIGINT (FK, nullable) | 제보자 user.id |
| status | VARCHAR(10) | PENDING / ACTIVE / REJECTED |
| created_at | DATETIME | 생성일 |
| updated_at | DATETIME | 수정일 |

인덱스: (lat, lng) 복합 인덱스 (반경 검색 최적화)
